Step 01 · Audit

Deep Analysis — What's Missing

The site is aesthetically strong. Playfair + Cormorant + forest/amber palette is a genuine premium signal. But it is currently a beautiful brochure, not a system.

What the Site Does Well

✅

Brand Identity

Forest/cream/amber palette, serif typography, and noise texture create a premium artisan feel that differentiates from mass-market brands.

✅

Trust Signals

Trust bar with "No SLS · No Parabens", handcrafted batches, 1,000+ customers, Made in India. These are the right inputs.

✅

WhatsApp CTA

Floating WhatsApp widget is the single most important retention asset on the page. Already present — just completely under-leveraged.

✅

Product Signals

Ingredient chips on hover, price + MRP discount display, star ratings, and product categories are all solid commerce fundamentals.

Critical UX Gaps

🚨

No User Identity Layer

Zero login, zero account, zero memory of the user. Every visitor is a stranger. You can't build retention without identity.

🚨

No Routine Context

Products are listed as isolated items — soap, lip balm, cream. There is no concept of "your morning routine" or "your haircare stack." No stacking logic exists.

🚨

No Post-Purchase System

What happens after someone orders? Nothing. No follow-up, no refill nudge, no routine check-in. The relationship ends at checkout.

🚨

No Subscription / Refill Mechanism

A ₹50 soap lasts ~30 days. With 1,000+ customers, there is a massive predictable refill opportunity being entirely ignored.

🚨

No Skin Profile / Personalization

A new user lands on the page and sees all products equally. No quiz, no filter by concern, no "products for oily skin" pathway.

🚨

Checkout Dead-End

All product links go to the old Wix site. The brand experience breaks entirely at the most critical conversion moment. This is a catastrophic leakage.

🚨

No Urgency / Behavioral Triggers

No "only 12 left," no "37 people viewing this," no limited-batch messaging. Handmade products are naturally scarce — this isn't communicated.

🚨

Newsletter is Inert

Footer email input exists but leads nowhere. No automation, no welcome sequence, no value hook to justify signing up.

Step 02 · Product Layer

The New Product System

Four interlocking engines that turn one-time buyers into routine-driven repeat customers.

Engine 1 — Routine Builder

The core concept: every product on the site belongs to a step in a daily routine. The system helps users build their personal 2–4 step routine and then automates refills for that routine.


How it works in the UI:

  • A "Build Your Routine" quiz (3–5 questions: skin type, concern, gender, preference) lives above the fold on mobile, in a persistent sidebar on desktop
  • Output: a personalized "Your Daily Ritual" stack — e.g., Marigold Mountain Soap (AM cleanse) + Café Vanilla Scrub (PM exfoliation) + Shea Moisture Cream (seal)
  • Each routine is saveable (via phone number / WhatsApp OTP — no email friction)
  • On product pages, show "Completes Your Routine" module — showing what the user is missing from their saved stack
  • Routine step labels added to product cards: "Step 1: Cleanse", "Step 3: Moisturise" — no redesign, just a small chip above the product name
Design Rule

The Routine Builder must feel like a skincare consultation, not a product filter. Every question should be phrased in terms of the customer's life, not the product catalog. "How does your skin feel by 3pm?" not "Select skin type."

Engine 2 — Smart Reorder System

Handmade soaps last ~30 days. Lip balms last ~45 days. Creams last ~60 days. These are predictable. Use them.

1-Click Reorder UX:

  • WhatsApp message arrives with product image, name, and a single reply "1" to reorder" — no login, no cart, no friction
  • Alternatively, a saved routine page (accessible via a short link sent post-purchase) has a single "Reorder My Routine" button
  • Pre-fill address from last order. One tap to confirm. Done.
  • Introduce "Subscribe & Save": commit to a 3-month or 6-month cycle, get 10% off and free shipping. This converts reorder customers into subscribers.

Engine 3 — Behavior Tracking Layer

You don't need expensive analytics. You need intentional data collection at 5 key moments:


EventData CapturedWhat It Unlocks
Skin Quiz CompletedSkin type, concern, age range, genderPersonalized product recommendations for lifetime
Product Page Viewed (3+ sec)Product ID, category, timestampInterest signals → WhatsApp follow-up nudge
Add to Cart / Add to RoutineProduct, quantity, routine positionRoutine completion data, upsell opportunity
Purchase CompletedOrder, delivery date, phone numberStart refill countdown timer
WhatsApp Reply"1" reorder / "PAUSE" / "SKIP"Behavioral signal for churn prediction

Store all of this in a simple Airtable or Supabase table keyed by phone number. This is your customer database. It replaces CRM for the early stage.

Engine 4 — Personalization Engine

Personalization at Bighi Brothers doesn't need ML. It needs 3 rules applied consistently:

  • Rule 1 — Skin Profile Match: If quiz says "oily skin + acne concern" → always surface Neem/Turmeric products first
  • Rule 2 — Purchase History: If customer bought Soap only → show Scrub as "Complete your cleansing routine" in every touchpoint
  • Rule 3 — Seasonal: In summer (Apr–Jun) → push Cooling/Mint variants. In winter (Nov–Feb) → push Rich Shea/Cocoa Butter products. This is a 2-line config change monthly.

Upgrade path: Once you have 500+ orders, run a simple collaborative filter — "Customers who bought X also bought Y" — using Airtable formulas or a basic Python script on purchase history.

Routine Adherence Tracking

Simple, not complex. Track 3 signals only:

  • Signal 1 — Reorder rate: Did they reorder within 40 days of last purchase? Yes = "Active." No = "At-risk."
  • Signal 2 — WhatsApp engagement: Did they reply to the Day 14 check-in? Engaged = healthy. Silent = send win-back.
  • Signal 3 — Routine completion: Do they own all 3 routine steps? If only 1 product, they're "Incomplete Routine" — a massive upsell opportunity.

A simple Airtable dashboard with these 3 columns gives you a full retention picture for every customer. Color-code: Green (reordered), Yellow (due), Red (churned).

After user scrolls past 60% of homepage, a sticky bottom bar appears: "Your routine is waiting — 3 products, ₹148/month." Dismiss button available.

Security & Compliance

Essential Security Measures

  • Enable SSL/TLS on all pages (GitHub Pages + Cloudflare provides this)
  • Never store raw payment data — Razorpay handles PCI compliance
  • Encrypt customer phone numbers in Airtable using field-level encryption
  • Implement rate limiting on WhatsApp API to prevent abuse
  • Add reCAPTCHA to checkout forms to prevent bot submissions
  • Regular security audits: monthly dependency check via npm audit

Privacy Compliance (India)

  • Add privacy policy page with data collection disclosures
  • Implement explicit WhatsApp consent checkbox (pre-checked, with opt-out)
  • Add "Request my data" functionality in WhatsApp bot
  • Set data retention policy: customer data deleted after 3 years of inactivity
  • Document data processing agreements with Interakt and Make.com

Performance Optimization

Frontend Performance Targets

  • Largest Contentful Paint (LCP): < 2.5 seconds
  • First Input Delay (FID): < 100ms
  • Cumulative Layout Shift (CLS): < 0.1
  • Total page weight: < 500KB (excluding images)
  • Image optimization: WebP format with JPEG fallback, lazy loading below fold

CDN & Hosting Enhancements

Backup & Disaster Recovery

Backup Strategy

  • Airtable: Daily automated backup to Google Drive
  • Razorpay: Transaction data retained for 7 years (regulatory requirement)
  • WhatsApp messages: Make.com stores scenario run history for 30 days
  • GitHub: Full code repository history with branch protection
  • Local backup: Monthly export of customer data to encrypted ZIP

Disaster Recovery Plan

  • Recovery Time Objective (RTO): 4 hours — site re-deployable via GitHub Actions
  • Recovery Point Objective (RPO): 24 hours — Airtable daily backup
  • Single point of failure: WhatsApp API — maintain Interakt account with backup operator
  • Payment processing: Razorpay fallback to UPI direct (manual)

10. Admin Panel Technical Architecture

Frontend Stack
  • Framework: React.js or Vue.js (SPA for smooth experience)
  • UI Library: Tailwind CSS (consistent with main site design)
  • Charts: Chart.js or Recharts (for analytics visualization)
  • Tables: React Table or TanStack Table (sorting, filtering, pagination)
  • Rich Text: TipTap or Quill (for CMS editor)
Backend & Database
  • API: Supabase or Firebase (serverless, real-time)
  • Authentication: Supabase Auth with JWT tokens
  • Database: PostgreSQL (via Supabase) — relational data for orders, customers
  • Storage: Supabase Storage for product images and documents
  • Real-time: Live order updates via WebSockets
Integrations
  • Payments: Razorpay Dashboard API for refunds and settlements
  • WhatsApp: Interakt API for message templates and broadcasts
  • Shipping: Shiprocket / Delhivery API for label generation and tracking
  • Email: Brevo API for transactional emails
  • SMS: Twilio or Exotel (backup to WhatsApp)
